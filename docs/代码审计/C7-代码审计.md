### 问题3：OnAppModeChanged 冗余 if/else 分支（⚠️偏差-3）

| 项目 | 内容 |
|:---|:---|
| **审查路径** | `src/DirectoryCleanAgent/ViewModels/QuarantineViewModel.cs:649-665` |
| **如何发现问题** | 审查 AppMode 变更处理时发现 if/else 两条分支包含完全相同的日志+刷新逻辑，仅 `Dispatcher.Invoke` 包裹与否有区别 |
| **影响范围（风险等级）** | 极低 — 功能正常，纯代码清洁度问题 |
| **思考修复方案（权衡）** | `Application.Current?.Dispatcher.Invoke` 使用 null 条件运算符 `?.`，若 Dispatcher 为 null 则整体不执行。提取公共 log 到外层，Invoke 内部仅调用 `RefreshCommandStates()`。单元测试环境 Dispatcher 为 null 时走 fallback |
| **修复方案解析（代码对比）** | **前**：`if (Dispatcher != null) { Invoke(() => { Log; Refresh; }); } else { Log; Refresh; }`；**后**：`Log; Dispatcher?.Invoke(() => RefreshCommandStates()); if (Dispatcher == null) RefreshCommandStates();`，消除 log 重复 |

### 问题4：HasSelection O(n) 全量遍历（❓待确认-4）

| 项目 | 内容 |
|:---|:---|
| **审查路径** | `src/DirectoryCleanAgent/ViewModels/QuarantineViewModel.cs:176` |
