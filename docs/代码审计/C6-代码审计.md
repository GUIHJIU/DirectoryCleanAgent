### 2.2 偏差2 + 缺失1：OnExit 未显式刷新数据队列（⚠️偏差 + ❌缺失）

| 审查路径 | `src/DirectoryCleanAgent/App.xaml.cs:803-825` |
| :--- | :--- |
| **如何发现问题** | 实现细节文档**假设5**明确指出"App.OnExit 未调用 IAuditLogRepository.FlushAsync()"。深入追踪调用链：`App.OnExit → DI容器Dispose → AuditLogRepository.Dispose → BatchWriteQueue.Dispose → 最终FlushAsync`。发现隐式 Dispose 链确实会触发刷新（`BatchWriteQueue.Dispose:208-234`），但依赖两个隐式前提：① DI容器正确按注册逆序释放 Singleton；② `ISqliteConnectionFactory` 在 `IAuditLogRepository` 之后释放以保证最终 flush 的 SQL 写入有效。 |
| **影响范围（风险等级）** | 中 — 数据不丢失（Dispose 链保障），但显式调用缺失降低了代码意图的可读性。若未来重构 DI 注册顺序或改变对象生命周期，可能引入静默数据丢失。 |
| **思考修复方案（权衡）** | 方案A（选用）：在 `Dispose()` 前显式调用 `FlushAsync`，形成"显式预刷新 + Dispose 链二次保障"的双重安全模式。`BatchWriteQueue.FlushCoreAsync` 在队列为空时直接返回（`batch.Count==0`），双重调用幂等安全。方案B（否决）：仅在 Dispose 后调用 — 不可行，因为 `ISqliteConnectionFactory` 可能已随容器释放。方案C（否决）：移除 Dispose 中的 Flush，仅依赖显式调用 — 违反防御性编程原则。 |
| **修复方案解析（代码对比）** | **修改前**：`EverythingDependencyDetector.Cleanup()` 后直接 `disposable.Dispose()`。**修改后**：在两者之间插入 try-catch 包裹的显式刷新块，先调用 `IAuditLogRepository.FlushAsync` 和 `IDeletionRecordRepository.FlushAsync`，再释放容器。异常被捕获并记录警告，不阻塞退出流程。 |

### 2.3 偏差3：审计日志分页总数估算不精确（⚠️偏差）

| 审查路径 | `src/DirectoryCleanAgent/ViewModels/AuditLogViewModel.cs:223-231` + `IAuditLogRepository.cs` |
| :--- | :--- |
| **如何发现问题** | 审查 `ExecuteSearchAsync` 的分页逻辑：当第一页返回恰好 50 条时，`TotalCount = (0 * 50) + 50 = 50`，`TotalPages = 1 + 1 = 2`。实际可能有数百条记录，但 UI 仅显示"第1/2页，共50条"，用户必须翻到最后一页才知道真实总数。`IAuditLogRepository` 接口未提供 COUNT 查询能力。 |
| **影响范围（风险等级）** | 低 — 不影响数据正确性，仅影响分页导航的用户体验。当记录数 >50 时总页数显示偏低。 |
| **思考修复方案（权衡）** | 方案A（选用）：扩展 `IAuditLogRepository` 接口，新增 `QueryCountAsync` 方法，在 `AuditLogViewModel.ExecuteSearchAsync` 中与 `QueryAsync` 并行调用。方案B（否决）：在 QueryAsync 返回的 SQL 中同时执行 COUNT OVER 窗口函数 — 每条记录携带额外字段，浪费带宽。方案C（否决）：接受估算 — 用户翻到最后页才修正，体验差。选择方案A：接口变更虽涉及3个文件，但改动集中、语义清晰、性能无损（COUNT 查询走索引，<1ms）。 |
| **修复方案解析（代码对比）** | **接口**：新增 `Task<int> QueryCountAsync(...)` 方法签名，参数与 `QueryAsync` 一致。**实现**：复用相同的参数化 WHERE 构建逻辑，执行 `SELECT COUNT(*) FROM AuditLog`。**调用方**：`entriesTask` 和 `countTask` 并行执行（`Task.WhenAll`），`TotalCount = countTask.Result`，`TotalPages = (int)Math.Ceiling((double)totalCount / PageSize)`。 |

### 2.5 待确认B：FreedBytes=0 语义误导（❓待确认 → 已修复）
