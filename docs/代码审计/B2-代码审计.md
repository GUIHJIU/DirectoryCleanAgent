- **规则优先级验证**：`HeuristicRuleLoader.Load()` 现已验证优先级，配置文件中的无效优先级会产生警告日志。建议在 C4（设置页面）中添加规则校验 UI，在用户编辑规则 JSON 后实时提示优先级错误（而非等到加载时静默过滤）。
- **IExclusionManager 扩展**：B1（流式适配器）应在扫描管道第 K 步调用 `IExclusionManager.IsExcluded()` 做前置过滤，避免排除目录中的文件流经完整的规则评估管道，减少不必要的 CPU 开销。
- **预存在集成测试失败**：`DecisionEngine_Arbitrate_WithoutAI_SuggestDelete_ReturnsManualReview` 为 B3 预存在问题（`SuggestDelete + AI禁用 → 期望 ManualReview，实际 SuggestDelete`），应在 B3 审计时修复。
- **`BuiltInForbidRules.NameForbidRule` 路径匹配**：当前 `FileNameForbidRule` 的目录匹配逻辑（使用 `EndsWith` + `StartsWith` 组合）未使用 `PathNormalizer`，与本次修复的 `PathForbidRule` 不一致。建议在后续维护中统一标准化。
- **`BuiltInProtectRules.PathProtectRule` 的 `Contains` 匹配**：当前使用 `filePathUpper.Contains(protectedPath)` 做子串匹配，与 Glob 修复的方向一致——建议改为路径组件级匹配（如 `TryMatchPathComponent`），避免 `\Docker` 匹配 `\NotDocker\` 之类的边缘误报。
