### 2.1 ⚠️偏差1：InverseBoolConverter 误用于 int 类型导致警告文本永不显示

| 审查路径 | 如何发现问题 | 影响范围（风险等级） | 思考修复方案 | 修复方案解析 |
| :--- | :--- | :--- | :--- | :--- |
| `SettingsWindow.xaml:155` → `InverseBoolConverter` | 代码审查：`InverseBoolConverter.Convert` 仅处理 `value is bool`，绑定到 `IncludedVolumes.Count`（int）时永远返回 `false`→`Collapsed`，导致"未检测到NTFS卷"警告永远不显示 | **高** — 功能性Bug。当用户磁盘无NTFS卷时，UI无任何提示，用户无法理解为何卷列表为空 | **方案A（采用）**：新建专用 `CountToVisibilityConverter`，语义清晰、复用价值高、不修改现有Converter行为。**方案B（否决）**：在ViewModel添加 `HasNoVolumes` 属性——改动量大且仅此一处使用。**方案C（否决）**：修改 `InverseBoolConverter` 兼容 int——改变现有语义可能影响其他绑定 | ```csharp\n// 新建 CountToVisibilityConverter\n// count==0 → Visible（空列表警告）\n// count>0 → Collapsed\n// ConverterParameter="Inverse" 反转\npublic class CountToVisibilityConverter : IValueConverter\n{\n    public object Convert(object value, Type targetType,\n        object parameter, CultureInfo culture)\n    {\n        int count = value is int i ? i : 0;\n        bool inverse = parameter is string s\n            && s.Equals("Inverse", ...);\n        bool visible = inverse ? count > 0 : count == 0;\n        return visible ? Visibility.Visible\n            : Visibility.Collapsed;\n    }\n}\n```\n```xml\n<!-- XAML: 替换前 -->\nVisibility="{Binding IncludedVolumes.Count,\n  Converter={StaticResource InverseBoolConverter}}"\n<!-- 替换后 -->\nVisibility="{Binding IncludedVolumes.Count,\n  Converter={StaticResource CountToVisibilityConverter}}"\n``` |
