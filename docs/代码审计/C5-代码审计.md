## 5. 预防措施与后续 Action

### 5.1 代码规范补充

| 规范 | 说明 |
|------|------|
| 错误统计暴露 | 所有带有逐项异常隔离的批处理方法，必须在返回值中包含失败计数/SkippedCount，不可仅写日志 |
| 接口进度参数 | 所有耗时 > 5 秒的异步枚举方法（`IAsyncEnumerable<T>`），必须提供 `IProgress<?>?` 参数，避免用户面对"假死"状态 |
| 进度节流统一 | 同一管道内的进度报告使用一致节流间隔。建议提取共享常量到 Core 项目 |
| 注释-实现一致性 | 接口 XML 注释是契约的一部分——实现变更时必须同步更新接口注释 |
