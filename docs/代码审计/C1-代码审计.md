## 2. 问题详情与修复

### 2.1 偏差2：WarningBanner 颜色硬编码，深色主题下不协调

| 项目 | 内容 |
|:---|:---|
| **序号** | 1 |
| **审查路径** | `src/DirectoryCleanAgent/Controls/WarningBanner.xaml:13-15` → `BannerBorder.Background` / `BorderBrush`；`:31` → `TextBlock.Foreground` |
| **如何发现问题** | 逐行审查 WarningBanner.xaml，发现三个颜色值（`#FFF8E1`、`#FFB300`、`#E65100`）直接硬编码在 XAML 属性中。同时检查主题文件 `LightTheme.xaml`（`:48-50`）和 `DarkTheme.xaml`（`:48-50`），发现已定义了对应的主题刷 `WarningBannerBackgroundBrush` / `WarningBannerForegroundBrush` / `WarningBannerBorderBrush`，但 WarningBanner 控件未引用 |
| **影响范围（风险等级）** | **中** — 浅色主题下无影响（硬编码值与 LightTheme 定义一致）；深色主题下警告横幅保持亮黄色（背景 `#FFF8E1` + 文字 `#E65100`），与深色窗口背景（`#1E1E1E`）形成刺眼对比，且文字色未适配深色背景，影响视觉一致性和用户体验 |
| **思考修复方案（权衡）** | 唯一方案：将硬编码色值替换为 `{StaticResource}` 引用。LightTheme 中定义的颜色与原始硬编码完全一致（`#FFF8E1` / `#FFB300` / `#E65100`），确保浅色主题零视觉变化。DarkTheme 中定义为深棕背景（`#3E2723`）+ 浅橙文字（`#FFB74D`），WCAG AA 对比度约 4.8:1，确保可读性 |
| **修复方案解析（代码对比）** | **修复前**：`Background="#FFF8E1"` / `BorderBrush="#FFB300"` / `Foreground="#E65100"` — 三种颜色全部硬编码，浅色主题正常但深色主题显示异常。**修复后**：`Background="{StaticResource WarningBannerBackgroundBrush}"` / `BorderBrush="{StaticResource WarningBannerBorderBrush}"` / `Foreground="{StaticResource WarningBannerForegroundBrush}"` — 颜色由当前主题 ResourceDictionary 动态提供，浅色主题效果完全不变，深色主题自动适配 |

| 项目 | 内容 |

| **序号** | 2 |
| **审查路径** | `src/DirectoryCleanAgent/MainWindow.xaml:17` → `<converters:AppStateToBoolConverter x:Key="AppStateToBoolConverter" />` |
| **如何发现问题** | 全量搜索 `src/` 目录下 `AppStateToBoolConverter` 的所有引用：Converter 类定义在 `Converters/AppStateToBoolConverter.cs`，测试覆盖在 `ConvertersTests.cs:119-145`（3个测试），XAML 注册在 `MainWindow.xaml:17`。但在所有 XAML 文件中搜索 `{StaticResource AppStateToBoolConverter}` 返回 0 结果 —— 注册了但从未在任何 Binding 中使用 |
