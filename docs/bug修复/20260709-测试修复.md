## 已修复项详情

### P0-1: SkipException → `[EverythingFact]` 自定义属性（~19 tests）

**根因**: `EverythingTestHelper.cs` 自定义 `SkipException` 类不被 xUnit 识别，环境不可用时测试标记为 Failed 而非 Skipped。

**修复**:
1. 创建 `EverythingFactAttribute.cs`：继承 `FactAttribute`，在构造函数中检查 `EverythingTestHelper.IsAvailable`，不可用时设置 `Skip` 属性
2. 更新 4 个测试文件（19 个测试方法）：`[Fact]` + `[Trait("Requires", "Everything")]` → `[EverythingFact]`
3. 移除所有 `EverythingTestHelper.SkipIfUnavailable(logger)` 调用
4. 删除 `SkipException` 自定义异常类

**影响文件**:
- `src/DirectoryCleanAgent.Tests/Infrastructure/EverythingFactAttribute.cs`（新建）
- `src/DirectoryCleanAgent.Tests/Infrastructure/EverythingTestHelper.cs`
- `src/DirectoryCleanAgent.Tests/Integration/StartupFlow/EverythingDetectionIntegrationTests.cs`
- `src/DirectoryCleanAgent.Tests/Integration/ScanAndEvaluate/EverythingScanIntegrationTests.cs`
- `src/DirectoryCleanAgent.Tests/Integration/StartupFlow/IndexNotReadyIntegrationTests.cs`
- `src/DirectoryCleanAgent.Tests/Integration/Cancellation/ScanCancellationTests.cs`

**结果**: 19 个 Everything 相关测试从 Failed → **Skipped**（正确行为）

### P0-2: ShellFileOperation 路径处理 + 错误码修复（~15 tests）

**根因**:
1. `SHFileOperationW` 对 `\\?\` 前缀路径兼容性不一致（不同 Windows 版本行为不同）
2. `Shell32Native.cs` 错误码常量使用 HRESULT 格式（0x80070020），但 `SHFileOperationW` 返回原始 Win32 错误码（32）
3. `OperationExecutor.ExecuteDelete` 缺少 `File.Delete` 回退机制（`BackupManager` 有但 `OperationExecutor` 没有）

**修复**:

1. **`ShellFileOperation.cs`**：
   - 4 个公开方法（`DeleteToRecycleBin`、`DeletePermanently`、`MoveFile`、`CopyFile`）入口处添加 `PathNormalizer.Denormalize()` 去规范化
   - `IsLockViolation` 同时检查 HRESULT 格式、Win32 原始错误码、以及 HRESULT 低 16 位
   - `GetErrorMessage` 同时映射两种格式的错误码

2. **`Shell32Native.cs`**：
   - 新增 Win32 原始错误码常量（`WIN32_ERROR_*`）
   - 新增 `ExtractWin32ErrorCode()` 辅助方法
   - 原有 HRESULT 格式常量保留（`SHQueryRecycleBinW` 仍使用 HRESULT）

3. **`OperationExecutor.cs`**：
   - `ExecuteDelete` 中添加 `File.Delete` 回退：当 `SHFileOperationW` 失败且非锁定冲突时，尝试 `System.IO.File.Delete`
   - 成功判断兼容两种错误码格式

4. **`DeleteExecuteIntegrationTests.cs`**：
   - 审计日志测试：查询条件从 `"Delete"` 修正为 `"FILE_DELETED"`（与生产代码一致）
   - 隔离区测试：补充 `ConstructQuarantineFileName` 和 `IsLargeFileForQuarantineAsync` 的 Mock 设置

**影响文件**:
- `src/DirectoryCleanAgent.Operations/Interop/ShellFileOperation.cs`
- `src/DirectoryCleanAgent.Operations/Interop/Shell32Native.cs`
- `src/DirectoryCleanAgent.Operations/OperationExecutor.cs`
- `src/DirectoryCleanAgent.Tests/Integration/ExecuteAndRollback/DeleteExecuteIntegrationTests.cs`

**结果**: DeleteExecute 9/9 通过，BackupManager 部分通过

### P1-1: DecisionEngine 测试预期更新（1 test）

**根因**: 测试期望 AI 禁用时 `SuggestDelete → ManualReview`，但设计文档 V3.7 决策表情况4 规定 AI 禁用时按规则裁决原值透传 → `SuggestDelete`。代码正确，测试预期过时。

**修复**: `ScanRuleEvaluateIntegrationTests.cs` — `Assert.Equal(FinalAction.SuggestDelete, ...)`，测试名更新为 `_ReturnsSuggestDelete`

---

### P1-2: FinalActionToDisplayConverter 无效输入修复（1 test）

**根因**: Switch 表达式将无效输入（如字符串 `"invalid"`）静默映射为 `FinalAction.ManualReview` → `"需确认"`，而非让外层 try-catch 返回 `"未知"`。

**修复**: `FinalActionToDisplayConverter.cs` — 默认分支改为 `throw new ArgumentException(...)`，触发外层 catch 返回 `"未知"`

---

### P1-3: WaitForIndex 取消测试类型匹配（1 test）

**根因**: `Assert.ThrowsAsync<OperationCanceledException>` 要求精确类型匹配。`Task.Delay` 取消时抛出 `TaskCanceledException`（继承自 `OperationCanceledException` 但不精确匹配）。

**修复**: `IndexNotReadyIntegrationTests.cs` — `ThrowsAsync` → `ThrowsAnyAsync`

---

### P2-1: FileListViewModel Dispatcher 安全模式（8/9 tests）

**根因**: `FileListViewModel` 有 11 处无保护的 `Application.Current.Dispatcher.InvokeAsync()` 调用。xUnit 测试运行器中 `Application.Current` 为 null，导致 NRE。

**修复**:
1. 添加 `RunOnUIThreadAsync(Action action)` 辅助方法：`Application.Current?.Dispatcher != null` 时使用 Dispatcher，否则同步执行
2. 替换 `RebuildGroupTreeAsync` 和 `LoadDataAsync` 中测试触发的 3 个 Dispatcher 调用点

**参考**: `QuarantineViewModel` 已有的 `Application.Current?.Dispatcher` null 守卫模式

**影响文件**: `src/DirectoryCleanAgent/ViewModels/FileListViewModel.cs`

---

### P2-2: 主题文件路径查找（2 tests）

**根因**: `FindProjectDir` 从测试输出目录向上查找项目源目录时路径层级不匹配，找不到 `LightTheme.xaml` 和 `DarkTheme.xaml`。

**修复**:
1. `.csproj` 添加 `<Content Include="..\DirectoryCleanAgent\Themes\*.xaml">` 将主题文件复制到测试输出目录
2. `FindProjectDir` 增加策略：优先检查输出目录中是否已有主题文件，回退时也检查 `Themes` 子目录

**影响文件**:
- `src/DirectoryCleanAgent.Tests/DirectoryCleanAgent.Tests.csproj`
- `src/DirectoryCleanAgent.Tests/Integration/StartupFlow/AppStartupIntegrationTests.cs`
