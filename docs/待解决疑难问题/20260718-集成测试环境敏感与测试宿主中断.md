# 集成测试环境敏感与测试宿主中断问题

- **发现日期**: 2026-07-18
- **状态**: 待处理
- **优先级**: 中(影响回归验证的可信度与可重复性)

## 现象汇总

2026-07-18 修复文件列表 bug 后做全量回归时,同一代码库多次运行结果不一致:

| 运行 | 总数 | 结果 | 备注 |
|------|------|------|------|
| 第 1 次全套 | 821 | 1 失败 | GroupNode 竞态(已修复) |
| 第 2 次全套 | **293** | exit 1,无失败明细 | **测试宿主疑似中途终止** |
| 第 3 次全套 | 870 | 3 失败 | 见下文 A/B + GroupNode |

### 问题 A:磁盘空间不足导致集成测试失败

`ScanRuleEvaluateIntegrationTests.ScanEvaluateDecide_RealFiles_ThroughPipeline`

```
System.IO.IOException : 磁盘空间不足。
: 'C:\Users\...\AppData\Local\Temp\scan_eval_...\Users\TestUser\Downloads\test_000040.zip'
   at TestFileGenerator.WriteFile(...)  (TestFileGenerator.cs:246)
```

`TestFileGenerator` 向 `%TEMP%`(C 盘)生成大量测试文件,对磁盘剩余空间敏感。
当日开发机 C 盘接近占满,测试直接 IOException 而非跳过。

### 问题 B:真实 Everything 取消测试的固有竞态

`EverythingScanIntegrationTests.EnumerateFilesAsync_RealEverything_CancellationStopsStream`
(EverythingScanIntegrationTests.cs:226 附近)

```
Assert.Throws() Failure: No exception was thrown
Expected: typeof(System.OperationCanceledException)
```

该测试依赖真实 Everything 服务:若查询结果流很短或消费很快,取消令牌
来不及在枚举循环内生效,枚举正常结束、无异常抛出。属环境状态相关的固有竞态
(Everything 索引量、是否正在重建索引都会影响)。

### 问题 C:测试宿主中途终止(未定位)

第 2 次全套运行仅执行 **293/821** 个测试即以 exit 1 结束,输出末尾却是
"已通过!"(仅统计了已执行部分),**没有任何失败明细或崩溃堆栈**。
疑似 test host 进程中途退出。当时 C 盘磁盘接近占满,可能与问题 A 同源
(临时文件写入失败导致 host 异常),但无直接证据。

### 问题 D:测试总数在运行间波动(821 vs 870)

同一代码库、同一命令,发现的用例总数不同,说明部分集成测试的发现/执行
依赖运行时环境条件。后果:两次运行的结果无法直接对比,回归信号不可靠。

## 建议处理方向

1. **TestFileGenerator 增加磁盘空间预检**:生成前检查目标卷剩余空间,
   不足时 `Skip`(xUnit 可用 `SkippableFact`)并给出明确原因,而非 IOException;
2. **RealEverything 取消测试**:注入慢消费者或确保结果流足够长
   (如查询保证多页结果),使取消窗口确定存在;或将断言放宽为
   "抛出 OCE 或枚举提前终止";
3. **环境依赖测试统一打 trait**(如 `[Trait("Category","RequiresEverything")]`
   / `RequiresDiskSpace`),本地与 CI 默认排除,专门的环境化流水线再跑,
   消除总数波动;
4. 问题 C 待复现:磁盘空间恢复后重跑全套若不再出现可关闭;若仍出现,
   用 `--diag` 收集 vstest 诊断日志定位 host 退出原因。

## 环境备注

发现当日开发机 **C 盘磁盘空间不足**(已当面提醒使用者清理)。
问题 A/C 在磁盘空间恢复前无法有效复验。
