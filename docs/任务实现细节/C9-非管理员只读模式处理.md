# 阶段性关键决策与细节控制文档 — C9 非管理员只读模式处理

---

#### 一、 关键架构/设计决策记录

1. **最终决策**：在已有的 MainViewModel/DeletionHistoryViewModel 权限守卫基础上，补全三个缺口：
   - MainViewModel.Initialize() 中直接字段赋值 `_isAdmin = _appStateService.IsAdmin` 完成启动同步
   - QuarantineViewModel 引入 `IAppStateService` 依赖，参照 DeletionHistoryViewModel 的守卫模式实现 `CanRestore`/`CanDelete` 属性和运行时守卫
   - QuarantineWindow.xaml 添加内联只读警告 Border（复用 MainWindow WarningBanner 的色值体系）

2. **核心权衡（Trade-off）**：
   - QuarantineViewModel 的 `OnAppModeChanged` 中使用 `Application.Current?.Dispatcher` 空检查，在单元测试环境下跳过 Dispatcher 直接执行。牺牲了测试环境对 UI 线程调度的 100% 仿真，换取代码在测试和生产环境的一致性运行。
   - MainViewModel.Initialize() 中直接设置字段 `_isAdmin` 而非通过 setter，避免了冗余的 `UpdateWarningBanner()`/`RefreshCommandStates()` 触发，但引入了少量字段级耦合（后续 Initialize 流程中的 `IsIndexReady = true` 会自然触发 UI 刷新）。

3. **C9审计修复 — QuarantineWindow 布局叠加**：QuarantineWindow.xaml 内层 Grid.Row="0" 中原三个 Border（容量信息栏、错误提示、只读警告）因同 Grid cell 叠加渲染，改为 StackPanel 垂直排列。修复仅涉及 XAML 布局结构，无逻辑变更。

4. **被否决的替代方案**：
   - **方案A**：在 QuarantineWindow 中复用 `WarningBanner` 用户控件。
     - 否决原因：WarningBanner 绑定到 `MainViewModel.WarningMessage`/`ShowWarningBanner`，QuarantineWindow 的 DataContext 是 `QuarantineViewModel`，需要额外的 DataContext 桥接或绑定路径穿越，增加 XAML 复杂度。
   - **方案B**：QuarantineViewModel 不引入 `IAppStateService`，而是通过构造函数传入 `bool isAdmin` 静态值。
     - 否决原因：无法响应运行时 AppMode 变更事件（虽然当前场景下 AppMode 不变，但架构设计上需要支持未来可能的权限提升场景）。

---

#### 二、 隐含前提与关键假设清单

* **[假设1]**：`WindowsPrincipal.IsInRole(WindowsBuiltInRole.Administrator)` 在目标 Windows 版本（Windows 10/11 家庭版/专业版/企业版）上能正确识别管理员权限。已知风险：某些域环境中本地管理员可能不被识别。
* **[假设2]**：应用程序启动后 AppMode 不会在运行时动态切换（即用户不会在不重启的情况下提升权限）。本实现支持 AppModeChanged 事件的理论响应，但实际触发路径仅存在于 `App.xaml.cs:DetectAdminRights`。
* **[假设3]**：`Application.Current` 在桌面 WPF 应用的正常启动流程中不会为 null。`OnAppModeChanged` 中的空检查是为了单元测试兼容性。

---

#### 三、 边界与范围界定

* **[明确不做]**：本期不实现运行时权限提升（UAC 重新提权后动态从 ReadOnly 切换到 Normal）。用户必须退出并以管理员身份重新启动。
* **[明确不做]**：本期不对非管理员用户隐藏删除/清理按钮，仅通过 `IsEnabled=false` 灰化处理。隐藏属于 UI 层面的附加需求。
* **[明确不做]**：本期不修改 DeletionHistoryViewModel 和 MainViewModel 中已有的 `Application.Current.Dispatcher.Invoke` 模式（与 QuarantineViewModel 新增的空检查模式不一致但属既有代码，非本任务范围）。

#### 四、 待确认事项

* **[待确认项A — ✅ 已修复]**：QuarantineViewModel 的 `ReadOnlyWarningMessage` 使用了硬编码的中文字符串 `"⚠️ 只读模式 — 请以管理员身份重新运行以启用清理功能（非管理员模式下数据分析功能仍可正常使用）"`。MainViewModel 中的 `UpdateWarningBanner()` 也使用了硬编码字符串。**已于 C9 代码审计中修复**：更新 `Strings.resx`/`Strings.en.resx` 中的 `Status.ReadOnlyMode` 和 `Warning.IndexingIncomplete` 键值，MainViewModel 和 QuarantineViewModel 均改为通过 `ILocalizationService.GetString()` 获取本地化文本。
